# Security Policy

未修正の脆弱性、credential、private host / inventory、秘密実値を public Issue や Pull Request に投稿しないでください。

GitHub の Security タブに private vulnerability reporting が表示される場合は、`Report a vulnerability` を使ってください。利用できない場合は、詳細を含めずに private な連絡経路を尋ねる Issue を作成してください。

報告には、可能な範囲で影響範囲、再現条件、期待した結果、実際の結果を含めてください。token、password、private key、秘密を含む raw log は添付しないでください。
